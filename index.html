WAI ARIA release history

  • WAI ARIA 1.0 became a W3C Recommendation in March 2014.
  • WAI ARIA 1.1 became a W3C Recommendation in December 2017.
  • WAI ARIA 1.2 became a candidate Recommendation as of 02 March 2021.

What are ARIA roles?

ARIA roles are HTML attributes that are added to elements using:

role="[role-type]"

ARIA roles can be used to add or change the semantic meaning of HTML elements in the accessibility tree. For example:

  • Is the element a menu?
  • Is the element a slider?
  • Is the element a spinner?
  • Is the element a progress bar?
  • Is the element a menu?

Changes to role categories

When WAI ARIA 1.0 became a W3C Recommendation in March 2014, there were four categories for roles.

  1. Abstract Roles
  2. Widget Roles and Widget Containers
  3. Document Structure Roles
  4. Landmark Roles

When WAI ARIA 1.1 became a W3C Recommendation in 14 December 2017, some additional categories were introduced, which meant thatsome existing 1.0 roles were moved to new categories.

  1. Abstract Roles
  2. Widget Roles
  3. Widget container Roles
  4. Document Structure Roles
  5. Landmark Roles
  6. Live Region Roles
  7. Window Roles
